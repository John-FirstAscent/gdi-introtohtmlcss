HTML

Let's look at the most basic HTML building blocks

Doctype

The first thing on an HTML page is the doctype, which tells the browser which version of the markup language the page is using.


						<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML
						4.01 Transitional//EN" "http://
						www.w3.org/TR/html4/loose.dtd">
						

						<!DOCTYPE html>

⚠️ Note: The doctype is not case sensitive.
DOCtype, doctype, DocType and DoCtYpe are all valid.

HTML Tag

The page must contain a set of HTML tags. All page content will be nested inside these HTML tags.

<!DOCTYPE html>
<html>

</html>

Head

The head of a page contains most of the page's initializations, rather than content.

This could include: Page title, external scripts, style sheets or even meta information.

Link Attributes

Links can have attributes that declare different actions on click, such as opening the link in a new tab or to send an email.


						Opens the home page in a new window or tab.
					

						E-mail us!

⚠️ Adding mailto: directly before an email address opens the user's default email application on click. By default, the subject line will automatically be filled with the email address.

Relative vs Absolute Paths
for links and images

Relative

Relative paths change depending upon the page the link is on (or the domain they are relative to)

Links within the same directory need no path information: filename.jpg

Subdirectories are listed without preceding slashes: images/filename.jpg

Absolute

Absolute paths refer to a specific location of a file, including the domain: http://www.girldevelopit.com/chap ters/detroit

Typically used when pointing to a link that is not within your own domain.

Lists

Both ordered and unordered lists utilize the li (list item) tag for new items.

⚠️ Note: Remember to keep track of nested opening and closing tags.
