Anatomy of a Website

A website is a way to present your content to the world. HTML is used to present that content while CSS is used to make it visually pleasing to a user.

What is CSS?

CSS stands for Cascading Style Sheets

CSS is a "style sheet language" that lets you style the HTML elements on your page. It cannot be used alone to build a website.

The CSS Rule

selector {
property: value;
}

A block of CSS code is called a rule.

The rule starts with a selector, then declares properties and values.

A property-value pair is called a declaration.

Embedded CSS

Embedded CSS is placed inside the <head>
element at the top of your HTML page.


						<!DOCTYPE html>
						<html>

						  <head>
						    <style type="text/css">
							p {
							  color: blue;
							  font-size: 16px;
							}
						    </style>
						  </head>

						  <body>
						    
This is a paragraph

						  </body>

						</html>

😖 However, because it is located in the HTML file, it cannot be shared by other HTML files.

Linked CSS

What if the CSS file was completely maintaned seperately from everything else and could be shared by all HTML files?

🎉 Voila! The linked CSS file!

Linked CSS

The link to the style sheet is placed in the <head> of the HTML file.


						<!DOCTYPE html>
						<html>

						  <head>
						    <link rel="stylesheet" type="text/css" href="style.css">
						  </head>

						  <body>
						  </body>
						</html>

Why we 💕 Linked CSS

👍 By placing a link to the shared CSS file on all HTML pages,
a single file can be managed on larger projects.

👍 It also saved on bandwith requests to the server when the page is loading.

👍 Linked CSS files are the preferred method for all nerds!

Colors

You can declare colors in many different ways.
It is up to you to pick which method you prefer.

Standard Color Name
red, lime, purple

Hexadecimal Value
#ff0000, #737373, #65b34e

RGB Value
rgb(255, 10, 20), rgb(17, 180, 17), rgb(0, 0, 0)

RGBa Value
rgba(255, 10, 20, 0.2), rgba(17, 180, 17, 1), rgba(0, 0, 0, 0.1)

HSL Value
hsl(0, 100%, 10%), hsl(1%, 10%, 90%), hsl(22%, 5%, 19%)

Declaring Colors

The color property changes the color of text.


						p {
						  color: red;
						}
						

						p {
						  color: rgb(255, 0, 0);
						}
						

						p {
						  color: #ff0000;
						}

Declaring Size

You can declare the size of elements in a few different ways.
It is up to you to pick which method you prefer.

Pixels (px)
the number of pixels declared is relative to the user's screen

Ems (em)
1em is equivalent to to the declared default size.
For example, if the default font size for a font is 20px, 0.5em would be 10px.

Percentage (%)
Generally used for delcaring responsive elements.

Declaring Fonts

All elements of a font can be manipulated:


						p {
						  font-family: Arial, sans-serif, Comic-Sans;
						  font-size: 18px;
						  color: #ff0000;
						  background-color: yellow;
						}

⚠️ Note: font-color is not a valid property (that always trips me up).

Declaring Font Size

The font-size property specifies the size of the font:


						p {
						  font-size: 12px;
						}
						

						p {
						  font-size: 1.5em;
						}
						

						p {
						  font-size: 100%;
						}

Using Shorthand

Some declarations can be written using shorthand:


						p {
						  font-style: italic;
						  font-weight: bold;
						  font-size: 10px;
						  font-family: Sand-serif;
						}
						

						p {
						  font: italic bold 10px sand-serif;
						}

Properties with Multiple Values

Some properties (such as fonts) can have
one or more comma separated values:


						p {
						  font-family: Arial, sans-serif, Comic-Sans;
						}

Which takes priority?

Your browser has a priority list of CSS depending on the selector:

  1. Important tag:
    
							color: red !important;
  2. Inline CSS: 
    
							style="color:red;"
  3. ID: 
    
							#footer {width: 100%;}
  4. Class: 
    
							.heavy {font-weight: bold;}
  5. Generic Element: 
    
							p {font-family: 'Helvetica';}

Cascading Priority

Your browser also assigns priority based on the specificity of the selection. The more specific selectors have higher priority:


						.main .sale .clearance p {
						  color: red;
						}
						.main .sale .clearance {
						  color: green;
						}
						.main {
						  color: purple;
						}

What if two selectors are identical?!

The tie-breaker is position in the CSS. Rules that are lower in the file overwrite rules higher on the file:


						a {
						  background-color: yellow;
						}
						a {
						  background-color: teal;
						}
						a {
						  background-color: black;
						}
